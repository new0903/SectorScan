### Управление сканированием

Запускает процесс сканирования 

http://localhost:2345/api/Stations/fullscan

Возвращает текущий результат сканирования

http://localhost:2345/api/Stations/stats

Останавливает процесс

http://localhost:2345/api/Stations/stop

отменяет текущий процесс в бд и останавливает задачу. При повтором запуске сканирования начнет с начала сканировать

http://localhost:2345/api/Stations/canceled

статистика по станциям 

http://localhost:2345/api/Stations/count



Есть GRPC запросы. 






##  ВАЖНО !!

в appsettings.json есть настройки

  "RequestSettings": {
    "MaxConnectionsPerServer": 30, //количество запросов
    "TimeoutSeconds": 60, //ожидание выполнения всех запросов
    "TimeoutUpdateProxy": 10, //обновление прокси раз в 10 минут
    "RandomStartRequestSeconds": 30 // время задержки перед запросом, что бы все запросы запускались не в один момент а с разным таймингом. Если значение больше TimeoutSeconds, то некоторые запросы будут завершаться даже не начавшись. Акуратно с этим параметром.  
  },

Не рекомендую делать больше 30 запросов. Могут спалить и прикрыть лавочку.

## Обзор

Загружает все станции в бд

Для сканирования использовал квадраты 6x6. Если в секторе больше 300 станций делим квадрат на 2 и снова и снова пока станций  не будет меньше 300.

Для ускорения я использовал прокси. 
Их необязательно использовать, но у ресурса есть лимиты. Сканирование тогда будет очень медленным.

Вот такой результат возвращаю

    public record QueryResult(
        string OperatorCode,//код оператора
        NetworkStandard Network,//тип сети 0=2g, 1=3g,2=4g,3=5g

        bool isDone=false) // это последнее сообщение
    {
        public DateTime Timestamp { get; init; } = DateTime.UtcNow; // дата
