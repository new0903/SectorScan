Отличный вопрос. Переход от взвешенного центроида к нелинейному методу наименьших квадратов (NLLS) — это переход от «среднего арифметического с весами» к настоящей геодезической задаче.

Суть метода

Вы ищете точку (x, y), чтобы сумма квадратов невязок была минимальной:

```
S = Σ (d_i - d̂_i)² → min
```

Где:

· d_i — измеренное расстояние до i-й вышки (через RSSI)
· d̂_i — евклидово расстояние от искомой точки (x, y) до координат вышки (x_i, y_i)

Упрощённый алгоритм (Гаусса–Ньютона)

```python
import numpy as np

def nlls_location(towers, max_iter=10):
    """
    towers = [
        {'lat': 55.123, 'lon': 37.456, 'rssi': -75},
        ...
    ]
    """
    # 1. Начальное приближение (ваш взвешенный центроид)
    x, y = weighted_centroid(towers)
    
    for _ in range(max_iter):
        # 2. Строим матрицу Якоби и вектор невязок
        J = []
        R = []
        
        for t in towers:
            # Расстояние от текущей точки до вышки
            dx = x - t['lon']
            dy = y - t['lat']
            d = np.sqrt(dx*dx + dy*dy)
            if d < 1: d = 1
            
            # Измеренное расстояние (по RSSI)
            d_measured = rssi_to_distance(t['rssi'])
            
            # Невязка
            r = d_measured - d
            R.append(r)
            
            # Строка Якобиана: производная d по x и y
            J.append([-dx/d, -dy/d])
        
        J = np.array(J)
        R = np.array(R)
        
        # 3. Решаем (J^T * J) * delta = J^T * R
        JTJ = J.T @ J
        JTR = J.T @ R
        
        # Защита от вырожденности
        if np.linalg.cond(JTJ) > 1e10:
            JTJ += np.eye(2) * 0.01
        
        delta = np.linalg.solve(JTJ, JTR)
        
        # 4. Обновляем координаты
        x += delta[0]
        y += delta[1]
        
        # 5. Проверка сходимости
        if np.linalg.norm(delta) < 1.0:  # меньше метра
            break
    
    return y, x  # lat, lon
```

Вспомогательные функции

```python
def rssi_to_distance(rssi_dBm):
    """RSSI → расстояние в метрах (для LTE 1800 МГц, город)"""
    if rssi_dBm > -60:
        return 50
    if rssi_dBm < -110:
        return 2000
    
    # Эмпирическая формула
    return 10 ** ((-rssi_dBm - 20) / 35)

def weighted_centroid(towers):
    """Ваш текущий метод — он же начальное приближение"""
    total_weight = 0
    sum_lat = 0
    sum_lon = 0
    
    for t in towers:
        # Вес пропорционален мощности сигнала
        w = 10 ** (t['rssi'] / 10)
        sum_lat += t['lat'] * w
        sum_lon += t['lon'] * w
        total_weight += w
    
    return sum_lat / total_weight, sum_lon / total_weight
```

Почему NLLS лучше центроида

Ситуация Центроид NLLS
Вышки с одной стороны Точка посередине (мимо) Смещается к вышкам
Одна вышка очень близко Точка почти на вышке Точка почти на вышке
Разбросанные вышки Среднее арифметическое Геометрический центр с весами
Ошибка в координатах вышки Съезжает пропорционально Устойчивее (итерации)

Весовая версия NLLS (ещё точнее)

Добавляем надёжность RSSI в матрицу Якоби:

```python
def weighted_nlls(towers):
    # Каждой вышке — вес из мощности сигнала
    weights = [10 ** (t['rssi'] / 10) for t in towers]
    W = np.diag(weights)
    
    # В формуле теперь: J^T * W * J * delta = J^T * W * R
    JTJ = J.T @ W @ J
    JTR = J.T @ W @ R
    # ... остальное как выше
```

Реалии для мобильного приложения

Минусы NLLS:

· Требует 3+ вышек (с 2 вышками вырождается)
· Медленнее центроида (10 итераций × 100 вышек = допустимо)
· Может расходиться при плохом начальном приближении

Практический совет:
Делайте 2-3 итерации NLLS после центроида — это даст 80% выгоды при 20% затрат.

Готовый компромисс (для вашего приложения)

```python
def hybrid_location(towers):
    # 1. Быстрый центроид
    lat, lon = weighted_centroid(towers)
    
    # 2. Если вышек >= 4 — уточняем NLLS (2-3 итерации)
    if len(towers) >= 4:
        lat, lon = nlls_location(towers, max_iter=3)
    
    return lat, lon
```

NLLS даст вам выигрыш в 20-40 метров в сложных сценариях (вышки в линию, один сильный сигнал). Для старта достаточно центроида — его вы уже правильно реализовали.
